d3.js is one of the best things that happened to data visualization on the web: full-featured, robust, and customizable...

As it is usually used in browser environments, there are some benefits in using it on servers. In the first article of this series, we saw why and how pre-rendering charts on the server was interesting.

The main goal is to do the heavy-lifting on the server in order to reduce page load time and other burden for the website visitors, while keeping all the power offered by d3.js, in terms of interactions in the browser.

In this follow-up post we'll expand the concept so that it's possible to update the chart in the client, by re-using the server-generated data binding on the client
